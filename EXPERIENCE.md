# Vibe Coding Experience Documentation

## Tool Selection Justification
I chose **Bolt.new** for this assignment. I selected it because it is a very powerful "vibe coding" tool that runs directly in the browser. It allows me to build a full-stack React application without needing complex environment setups. It felt like the most modern experience among the options.

## Development Process
The process was very conversational. I did not write the code line-by-line. Instead, I acted like a project manager.
1.  **Prompting:** I gave a clear instruction: "Create a Todo List App with React and Tailwind."
2.  **Generation:** The AI created all the file structures (components, styles, package.json) instantly.
3.  **Refinement:** The app worked on the first try. I checked the features, and the "Local Storage" feature was working perfectly.

## Challenges and Solutions
* **Challenge:** The main difficulty was knowing exactly how to describe the design I wanted in English.
* **Solution:** I used simple keywords like "Dark mode" and "Clean design." The AI understood these keywords well.
* **Fixes:** I did not have to fix much manually. The AI handled the logic for adding and deleting tasks correctly.

## Reflection
I was very surprised by the speed of **Bolt.new**. Usually, setting up a React project takes time (installing packages, setting up Tailwind). With vibe coding, it took less than 1 minute.
It changed my workflow because I focused more on "what" I wanted to build, rather than "how" to write the syntax. I will definitely use this tool for quick prototypes in the future. It makes coding feel much more accessible.
